# 技術文件審查清單

交付前用這份清單檢查文件品質。若是審閱別人寫的文件，也可直接依此輸出 review。

## 文件定位

- 是否明確標示文件類型：Tutorial / How-to / Explanation / Reference
- 是否清楚標示目標讀者
- 是否有單一且明確的文件目標
- 是否列出非目標，避免範圍失控

## 結構與可讀性

- 標題是否能讓讀者快速判斷內容
- 是否有清楚的章節層次
- 是否能快速掃讀
- 段落是否過長
- 是否應拆成多份文件而不是繼續堆內容

## 內容品質

- 是否有任務與概念混寫的問題
- 是否有多餘背景資訊干擾主要目標
- 是否補齊前置條件、限制與假設
- 是否有明確範例、步驟、表格或圖表支撐內容
- 是否涵蓋常見 gotchas 或例外情況

## 中文一致性

- 是否以中文為主要敘述語言
- 專有名詞是否保留必要英文並搭配中文說明
- 是否有不必要的中英混雜
- 是否出現冗長、官話或翻譯腔

## 正確性與維護性

- 指令、路徑、參數、欄位名稱是否正確
- 範例是否可執行或至少自洽
- 是否標示版本、日期、適用範圍或前提
- 是否有過期風險高但未標示假設的內容
- 是否應建立持續更新機制或納入 Definition of Done

## 審查輸出格式

若要輸出簡短 review，可使用：

```md
## 審查結果
- 文件類型是否清楚：
- 目標讀者是否清楚：
- 單一目標是否成立：
- 結構是否易讀：
- 是否混入非目標內容：
- 中文一致性是否合格：
- 正確性風險：
- 建議修改：
```
